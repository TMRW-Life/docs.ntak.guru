# Activate Reporting

This endpoint allows you to activate reporting on an accommodation by providing the accommodation `id` or `slug`.

## Requirements

- The accommodation must have a valid (live) certificate.
- The accommodation must have `reportingId` set.

Both of these above will be set, when you upload your `.CER` file, which is generated by the government's NTAK system, also `expireAt` property will be set automatically.

## Request

<Protocol method="POST" path="/v1/accommodations/{accommodation}/activate" />

<Tabs items={['cURL', 'PHP']}>
  <Tabs.Tab>
    [//]: # (@formatter:off)
    ```bash copy
    curl -X POST https://api.sandbox.ntak.guru/v1/accommodations/rqdsw/activate \
      -H "Authorization: Bearer {token}" \
      -H "Accept: application/json"
    ```
    [//]: # (@formatter:on)
  </Tabs.Tab>
  <Tabs.Tab>
    [//]: # (@formatter:off)
    ```php copy
    use TmrwLife\NtakGuru\Services\Accommodation;

    $gateway = Accommodation::setup([
      'accessToken' => {token},
      'isProduction' => false, // default: false
    ]);

    $response = $gateway->activate('rqdsw');
    ```
    [//]: # (@formatter:on)
  </Tabs.Tab>
</Tabs>

### URL Parameters

| Name            | Description                      |
|-----------------|----------------------------------|
| `accommodation` | The accommodation `id` or `slug` |
